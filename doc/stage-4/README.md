# Project Process and Resources

## Sprint Reviews
Sprint progress and completed features were reviewed informally during team meetings.  
Demonstrations of implemented functionality were performed within the development environment.

---

## Retrospectives
Retrospectives were conducted informally through team discussions during development.  
These discussions helped identify blockers, bugs, and possible improvements.

---

## Sprint Planning
Sprint planning was performed informally through team discussions.  
Tasks and priorities were defined collaboratively based on project milestones and current development needs.

---

## Source Repository
Project source code is hosted on GitHub:

https://github.com/AdelMej/portfolio-project

---

## Bug Tracking
Bugs were identified and resolved during development through manual testing and team discussions.  
Issues were tracked informally and resolved as they appeared during development.

---

## Testing Evidence and Results
Testing was performed manually by validating application behavior through the frontend, API responses, and database state.

Database inspection and debugging were performed using DBeaver to verify data integrity, query correctness, and system invariants.

Testing scenarios included:
- Session creation and management
- User session participation
- Payment handling and validation
- Database constraint verification

---

## Production Environment
The application was executed locally during development using a local PostgreSQL database instance and the backend API server.

The system was demonstrated in this environment during the project presentation.
